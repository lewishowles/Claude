# Central Agent Configuration Repository

Source of truth for Claude Code + OpenAI Codex CLI global configuration. Skills, hooks, and rules live here and are distributed to `~/.claude/` and `~/.codex/` via symlinks.

## Purpose

Single source for shared rules and skills. `scripts/sync.sh` composes agent-specific output files (`targets/claude/CLAUDE.md`, `targets/codex/AGENTS.md`) from content in `shared/`.

## Current vs target state

**Current (pre-Phase 3):** `CLAUDE.md`, `hooks/`, `skills/`, `settings.json` at repo root.  
**Target (post-Phase 3):** `shared/`, `targets/claude/`, `targets/codex/`, `skills/`, `scripts/`, `docs/`. See `.claude/PROGRESS.md` Phase 3.

## Symlink topology (after `setup-global.sh --both`)

```
~/.claude/CLAUDE.md         → repo/targets/claude/CLAUDE.md
~/.claude/settings.json     → repo/targets/claude/settings.json
~/.claude/skills/<name>     → repo/skills/<name>         (per-skill ×20)
~/.claude/hooks/<file>      → repo/targets/claude/hooks/<file>  (per-hook)
~/.codex/AGENTS.md          → repo/targets/codex/AGENTS.md
~/.agents/skills/<name>     → repo/skills/<name>         (per-skill ×20)
```

## Confirmed Codex behaviour

- **Global rules:** `~/.codex/AGENTS.md`
- **Project rules:** walks root → cwd for `AGENTS.md`; root real file or symlink covers both runtimes
- **Skills:** `~/.agents/skills/` (user scope); frontmatter `name`+`description` visible upfront; full skill loaded on use
- **Skill discovery:** description-driven, not slash-command; `Use this skill when` plus action-led wording and file globs in description = better matching
- **Hooks:** Codex hooks exist but parity is out of scope; skill descriptions carry discovery weight instead
- **Caveman + claude-mem:** already installed for Codex (caveman manually activated)

## Edit workflow

Edit source files here in the repo. Symlinks propagate immediately. After editing `shared/` content, run `scripts/sync.sh` to regenerate `targets/claude/CLAUDE.md` and `targets/codex/AGENTS.md`. Commit changes here.

Don't edit `~/.claude/` or `~/.codex/` directly — changes won't persist.

## Skill structure

Each skill: `skills/<name>/SKILL.md` with frontmatter (`name`, `description`, `related-skills`). Descriptions written for both Claude auto-trigger and Codex description-driven discovery — `Use this skill when` prefix, action-led wording, file globs inline.

## Adding a skill

1. Create `skills/<name>/SKILL.md`
2. Frontmatter: `name`, `description` (`Use this skill when` + action-led wording + globs), `related-skills`
3. Write content
4. Update `hooks/skill-autotrigger.sh` and `hooks/skill-file-trigger.sh` (or `targets/claude/hooks/` post-Phase 3)
5. Re-run `setup-global.sh` to create new symlinks
