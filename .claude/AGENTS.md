# Central Agent Configuration Repository

Source of truth for Claude Code + OpenAI Codex CLI global configuration. Skills, hooks, and rules live here and are distributed to `~/.claude/` and `~/.codex/` via symlinks.

## Purpose

Single source for shared rules and skills. `scripts/sync.sh` composes agent-specific output files (`targets/claude/CLAUDE.md`, `targets/codex/AGENTS.md`) from content in `shared/`.

## Confirmed Codex behaviour

- **Global rules:** `~/.codex/AGENTS.md`
- **Project rules:** walks root вҶ’ cwd for `AGENTS.md`; root real file or symlink covers both runtimes
- **Skills:** `~/.codex/skills/` for this machine's global shared skills; frontmatter `name`+`description` visible upfront; full skill loaded on use
- **Skill discovery:** description-driven, not slash-command; `Use this skill when` plus action-led wording and file globs in description = better matching
- **Hooks:** Codex hooks exist but parity is out of scope; skill descriptions carry discovery weight instead

## Edit workflow

Edit source files here in the repo. Symlinks propagate immediately. After editing `shared/` or `targets/<agent>/source/` content, run `scripts/sync.sh` to regenerate `targets/claude/CLAUDE.md` and `targets/codex/AGENTS.md`. Commit changes here.

Don't edit `~/.claude/` or `~/.codex/` directly вҖ” changes won't persist.

## Skill structure

Shared global skills: `skills/<name>/SKILL.md` with frontmatter (`name`, `description`, `related-skills`). Descriptions for shared skills are written for both Claude auto-trigger and Codex description-driven discovery вҖ” `Use this skill when` prefix, action-led wording, file globs inline.

## Adding a skill

1. Create `skills/<name>/SKILL.md`
2. Frontmatter: `name`, `description` (`Use this skill when` + action-led wording + globs), `related-skills`
3. Write content
4. Update `targets/claude/hooks/skill-autotrigger.sh` and `targets/claude/hooks/skill-file-trigger.sh`
5. Re-run `setup-global.sh` to create new symlinks
