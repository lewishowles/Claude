# Bash

## Config files

- Minimal comments, no headers
- `.env`/`.conf` concise, scannable
- Config organised cleanly

## Documentation

- Use `/path/to/directory` placeholders, not user-specific paths

## Styling

- Simple functions
- No premature abstraction
